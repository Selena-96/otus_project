# otus_project
Проект для курса "1С Архитектор"

Для проекта использована БСП версии 3.0.3.341. Работа велась в конфигураторе, все добавленные объекты имеют префикс "otus_".
Также все новые объекты включены в подсистему otus_Интеграция:

Описание объектов, добавленных в конфигурацию:
- Справочник otus_Соединения - настройка интеграции, в которомв шапке задаётся адрес подключения. а в табличную часть Состав добавляется детальная инормация в разрезе типов отправляемых/загружаемых объектов. Для выгружаемых данных должно быть указано условие выгрузки, для загружаемых нужно заполнять группу (консьюмер в Кафка).
- Регистр сведений otus_СообщенияКОтправке - содержит записи по каждому изменённому объекту для каждого топика, в который он будет отправлен.
  Объекты регистрируются в подписке на событие otus_РегистрацияИзмененийПриЗаписи.
- Регламентные задания otus_ОтправитьСообщенияKafka и otus_ПрочитатьСообщенияKafka запускаются для выгрузки записанных в регистр объектов и получения имеющихся в топике записей соответственно.
- Общий модуль otus_Интеграция содержит обработки перечисленных подписки и регламентных заданий; общий модуль otus_kafka содержит функции из обработки example.epf, идущей к ВК onec_librdkafka практически без изменений. ОМ otus_ОбщегоНазначения содержит некоторые дополнительные функции.

Все диаграммы находятся в папке doc/Схемы, где это возможно добавлены файлы puml, также все результаты в подпапке Скриншоты.
BPMN-схема проекта была создана с помощью StormBPMN.

# YAxUnit
В расширении YAxUnit 3 общих модуля, модуль ОМ_СозданиеОбъектов нужен только для подготовки тестовых данных для двух других: ОМ_РегистрацияОбъектовКОтправкеВKafka и ОМ_ОбменKafka, в которых стируются соответствено регистрация и обмен (отправка и получение сообщений в целом).

# Vanessa Automation
Сценарии тестирования находятся в отдельной папке doc/Сценарии тестирования. Фича-файл "00_01 ПодготовкаДанных" также служит для подготовки данных к корректному выполнению сценариев. Файл "01_01 ПроверкаРегистрацииДляИнтеграции.feature" содержит два теста: позитивный, в котором документ проходит по условию для регистрации к обмену, и негативный, в котором условие не выполняется, докумнт не регистрируется.
